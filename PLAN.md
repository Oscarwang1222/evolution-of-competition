# 模拟游戏开发计划

## 1. 项目概述

一个类似《信任的进化》的互动模拟游戏，核心玩法：
- 多种角色（卷王、跟风、创哥等）
- 每轮分配精力到"分数"或"创造力"
- 创造力通过乘数效应在后期放大分数
- 短轮数：卷王赢；长轮数：创造力角色赢

## 2. 技术栈

- **前端框架**: React 18 + TypeScript
- **构建工具**: Vite
- **测试**: Playwright（E2E 测试）
- **样式**: CSS（简单易维护）

## 3. 核心功能

### 3.1 角色系统
| 角色 | 描述 |
|------|------|
| 卷王 | 每轮 n 点全部用于分数 |
| 普通人 | 每轮 n-2 用于分数 |
| 跟风 | 每轮取平均值用于分数，剩余≥2用于创造力 |
| 超级跟风 | 每轮取平均值用于分数，剩余全部创造力 |
| 创哥 | 每轮 n-3 创造力，1 分数 |
| 顶级创哥 | 每轮 n-2 创造力，2 分数 |
| 均衡发展 | 每轮 (n-2)/2 分数，(n-2)/2 创造力 |
| 均衡卷王 | 每轮 n/2 分数，n/2 创造力 |
| 摆烂 | 每轮 1 分数，1 创造力 |

### 3.2 模拟引擎
- 初始化：设置角色列表、轮数、初始精力上限 n
- 每轮计算：
  1. 计算非跟风者的平均分数精力
  2. 根据角色规则分配精力
  3. 基础分数 × 1.05^创造力 = 实际分数
  4. 累积创造力
- 精力衰减：连续5轮每轮分数精力都<2，则 n-1

### 3.3 交互界面
- 角色选择页面（可多选）
- 配置：轮数、初始精力
- 结果展示：
  - 最终排名
  - 每轮分数曲线图

## 6. 测试用例

### 场景 1: 短期模拟
- 轮数: 10
- 初始精力: 10
- 角色: 卷王 vs 创哥
- 预期: 卷王赢

### 场景 2: 长期模拟
- 轮数: 50
- 初始精力: 10
- 角色: 卷王 vs 顶级创哥
- 预期: 顶级创哥赢

### 场景 3: 跟风机制
- 轮数: 20
- 角色: 卷王 + 跟风 + 超级跟风
- 验证跟风者的分数介于卷王和其他人之间
